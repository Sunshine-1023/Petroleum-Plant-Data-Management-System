# 采油廠油水井作業成本管理系統  
## 專案目的與環境配置運行文檔（Mac / Windows）

本文件提供本專案從 0 到可運行的完整說明，讓新同學、老師或評審可以在沒有口頭輔助的情況下，完成：

1. 環境安裝  
2. 資料庫初始化  
3. 前後端啟動  
4. 功能連通驗證

---

## 1. 專案目的

本專案以「采油廠油水井作業成本管理」為場景，完成一套前後端分離的資料庫應用系統，核心目的如下：

- 實現按單位代碼查詢業務資料與成本匯總
- 實現作業項目成本更新
- 演示資料庫高階能力：視圖、索引、存儲過程、觸發器、事務、游標、權限
- 提供可視化前端頁面與可驗證的後端 API

---

## 2. 專案技術棧

| 層級 | 技術 |
|---|---|
| 前端 | Vue 3 + JavaScript + Element Plus + Vite + Vue Router |
| 後端 | Django 4.2 + Django REST Framework + PyMySQL |
| 資料庫 | MySQL 8（資料庫名：`zyxt`） |
| 管理工具 | Navicat |
| 架構 | B/S 前後端分離 |

## 5. 環境變數配置

## 5.1 後端 `backend/.env`

可由 `backend/.env.example` 複製而來：

```env
# Django
SECRET_KEY=change-me-to-a-random-secret-key
DEBUG=True
ALLOWED_HOSTS=127.0.0.1,localhost

# MySQL（需與 init_mysql.sql 一致）
DB_NAME=zyxt
DB_USER=root
DB_PASSWORD=your_mysql_password
DB_HOST=127.0.0.1
DB_PORT=3306

# CORS
CORS_ALLOWED_ORIGINS=http://localhost:5173,http://127.0.0.1:5173
```

> 若課程驗收要求使用 `CYGLXT`，請把 `DB_USER/DB_PASSWORD` 換成對應帳號密碼。

## 5.2 前端 `frontend/.env`

```env
VITE_API_BASE_URL=/api
```

說明：Vite 會透過 `vite.config.js` 把 `/api` 代理到後端 `127.0.0.1:8000`。

## 6. 資料庫建立與初始化（所有平台通用）

請嚴格按順序執行：

1. 建立資料庫  
2. 導入實驗表與資料  
3. 安裝高階資料庫對象  
4. 驗證連線

## 6.1 建立資料庫 `zyxt`

使用 Navicat 執行 `scripts/init_mysql.sql`，或命令列執行：

```bash
mysql -u root -p < scripts/init_mysql.sql
```

## 6.2 導入業務表與測試資料

將你實驗 1~6 的 SQL 導入到 `zyxt`，核心表/視圖應至少包含：

- 表：`dept`、`well`、`project`、`material`、`material_detail`、`construct_unit`
- 視圖：`v_budget_status`、`v_mine1_projects`、`v_project_material`

## 6.3 安裝索引/存儲過程/觸發器/權限

在 `zyxt` 上執行：

```text
scripts/sql/04_advanced_objects.sql
```

此腳本為可重複執行（幂等）設計。

## 6.4 驗證資料庫準備完成

可執行以下 SQL 檢查：

```sql
SHOW TABLES;
SHOW TRIGGERS;
SHOW PROCEDURE STATUS WHERE Db='zyxt';
```

## 7. Mac 配置與啟動流程

## 7.1 安裝依賴

```bash
cd "/Users/sunshine/Desktop/数据库实践"
./scripts/setup.sh
```

終端 A（後端）：

```bash
cd "/Users/sunshine/Desktop/数据库实践/backend"
source ../.venv/bin/activate
python manage.py runserver
```

```bash
cd "/Users/sunshine/Desktop/数据库实践/frontend"
npm run dev
```

```bash
cd "/Users/sunshine/Desktop/数据库实践"
./scripts/start-dev.sh
```

## 8. Windows 配置與啟動流程（PowerShell）

## 8.1 安裝軟體

- Python 3.9+（安裝時勾選 Add to PATH）
- Node.js LTS
- MySQL 8
- Navicat

檢查版本：

```powershell
python --version
pip --version
node --version
npm --version
```

## 8.2 建立與啟用虛擬環境

```powershell
cd "C:\你的路径\数据库实践"
python -m venv .venv
.\.venv\Scripts\activate
pip install -r .\backend\requirements.txt
```

## 8.3 安裝前端依賴

```powershell
cd .\frontend
npm install
cd ..
```

## 8.4 啟動服務（兩個終端）

終端 A（後端）：

```powershell
cd "C:\你的路径\数据库实践\backend"
..\.venv\Scripts\activate
python manage.py runserver
```

> 若上面路徑格式不適配，改用：
>
> ```powershell
> cd "C:\你的路径\数据库实践"
> .\.venv\Scripts\activate
> cd .\backend
> python manage.py runserver
> ```

終端 B（前端）：

```powershell
cd "C:\你的路径\数据库实践\frontend"
npm run dev
```

## 10. 常見問題排查

## 10.1 `Can't connect to MySQL`

- 檢查 MySQL 是否啟動
- 檢查 `backend/.env` 帳密/端口
- 檢查 `DB_NAME` 是否為 `zyxt`

## 10.2 前端能開，但 API 報錯

- 後端未啟動或端口不是 8000
- `vite.config.js` 代理未對準 `127.0.0.1:8000`

## 10.3 成本更新提示「只能在工作時間」

資料庫觸發器 `trg_project_update_time_restrict` 有時間限制。  
可在工作時間演示，或臨時停用此觸發器。

## 10.4 觸發器演示時 `MaterialCost` 不同步

先執行 `scripts/sql/04_advanced_objects.sql`，確認存在：

- `trg_material_detail_update_project_cost`
- `trg_material_detail_update_project_cost_upd`
- `trg_material_detail_update_project_cost_del`
